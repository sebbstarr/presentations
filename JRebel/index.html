JRebel

Reload code changes instantly

Created by Sebastian Trotz

Vorstellung

Hintergrund

  • Firma ist ZeroTurnaround
    • 2008 gegründet von Jevgeni Kabanov and Toomas Römer
      • Kabanov's Ph.D. Thema war "Towards a more productive Java EE ecosystem"

  • Vertreibt JRebel als Java IDE Plugin
    • Build- und Redeploy-Phasen im Java EE Entwicklungszylus werden vermieden

IDE Integration

Eclipse-Plugin

  • Im Eclipse-Marketplace herunterladen
    • es gibt einen neuen View
    • lässt sich sehr einfach konfigurieren

  • Es ist möglich JRebel 30 Tage auszuprobieren

Bestandsaufnahme

Entwicklung

  • Serverstart dauert 2-3 min mit minimalen Produktdaten
    • verlängert sich abhängig von Grösse der Produkt-XML-Datei
    • mit Backoffice-Intialisierung dauert es fast 15 Minuten

  • Probleme mit Serverneustarts
    • Entwickler geriet aus dem Flow
    • Entwickler muss sich anderweitig beschäftigen

Featureliste - Teil 1

Features

  • Changes to method bodies
  • Adding/removing Methods
  • Adding/removing constructors
  • Adding/removing fields
  • Adding/removing classes
  • Adding/removing annotations

Featureliste - Teil 2

Features

  • Changing static field value
  • Adding/removing enum values
  • Modifying interfaces
  • Replacing superclass
  • Adding/removing implemented interfaces
  • Initializes new instance fields

JRebel Spring plugin

Spring Integration

  • reload Spring dependencies
  • registering new Spring beans
  • adding/removing dependencies
  • adding new MVC controllers and handler methods
  • These can be done using either XML configuration or annotations

Was es uns bringt

Vorteile

  • keine unnötigen Unterbrechungen
    • neue Funktion wurde hinzugefügt und soll getestet werden

  • schnellere Refactorings
    • Test der reactorisierten Funktionalität vorher und nachher

  • spart Zeit
    • die kann dann für wichtigere Tehmen (z.B Schreiben von Unit-Tests) investiert werden
